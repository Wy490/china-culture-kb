三种架构方案对比

选择最适合MVP的知识库Agent架构

A

MCP Server + Claude Code

构建MCP服务器提供知识库专用工具（采集、核查、入库、匹配、生成），通过Claude Code调用

优势

  • 零前端开发，直接用Claude Code交互
  • Claude Code自带web fetch、文件读写能力
  • MCP工具可复用、可扩展
  • 最贴近MVP"先跑起来"的思路

劣势

  • 依赖Claude Code环境运行
  • 非技术用户需要学习CLI
B

Claude Agent SDK 自定义Agent

用Anthropic Agent SDK构建独立Agent程序，自定义工具集和对话流程

优势

  • 完全自定义Agent行为和对话流程
  • 可独立运行，不依赖Claude Code
  • 后续可包装为API或Web服务

劣势

  • 开发量较大（需要构建整个Agent框架）
  • Agent SDK尚在演进中
  • 需要自己处理对话上下文管理
C

独立Python CLI

Python命令行工具，用Anthropic SDK调用Claude做分析，subcommands管理各功能

优势

  • 完全自主，无外部Agent框架依赖
  • 可用Python生态（yt-dlp、BeautifulSoup等）
  • subcommands清晰：fetch/verify/match/generate

劣势

  • 对话交互体验不如Agent模式
  • 需要自己写所有胶水代码
  • 扩展为Web需要大量重构

推荐：方案A（MCP Server + Claude Code）

最快实现MVP，充分利用Claude Code现有能力，只补知识库专用逻辑
