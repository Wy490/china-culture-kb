# 知识库条目撰写规范

## 1. 来源等级

新增或引用来源时，优先在条目 `### 来源` 内标注等级：

- A级：正史、一手文献、原始档案、本人著作、考古报告、正式法律或行政文书。
- B级：地方志、官方展陈、官方文旅资料、国家级/省级名录、文保单位资料。
- C级：学术专著、论文、专业工具书、权威百科、二手研究。
- D级：民间口述、地方传说、戏曲改编、旅游讲述、未能定位原始出处的公开材料。

传说、志异、鬼怪故事如果主要依赖 C/D 级材料，`### 可信度` 必须写为 `待核实` 或 `混合标注`，不得写成确证历史。

## 2. 知识库标签

条目可在基础字段后增加以下可选机器字段：

```md
- **knowledge_domain**：core_china_culture
- **entry_role**：core_entry
- **era**：宋
- **asset_usage**：character_clothing、character_props、scene_space、scene_props、story_motif、credibility_boundary、gears_delivery
```

常用取值：

- `knowledge_domain`：`core_china_culture`、`era_setting`、`regional_culture`、`folklore_zhiyi`、`gears_asset`
- `entry_role`：`core_entry`、`setting_pack`、`motif_pack`、`asset_pack`、`regional_pack`

## 3. 资产拆分

面向故事生成和 GEARS 供稿的条目，必须尽量补以下小节：

```md
### 人物

- 人物名或角色类型：身份、可信度边界、适用场景

### 场景

- 场景名：空间类型、时代边界、是否为自然/建筑/室内场景

### 人物随身道具

- 道具名：随人物移动，或作为角色识别物

### 场景陈设

- 陈设名：固定在场景中，不作为人物随身物
```

原则：

- 人物、服装、随身道具、场景、场景陈设必须分开。
- 洞穴、书院、官署、祠庙、街巷、渡口等是场景资产。
- 书卷、手稿、印章、伞、刀、灯笼等随人物移动时是人物随身道具。
- 书桌、书架、案卷、灯盏、香炉、岩壁、石阶等固定在空间内时是场景陈设。
- 同一物件在不同场景中可能改变归属，必须按故事镜头实际用途标注。

## 4. 朝代服饰、建筑与器物条目细化标准

所有朝代服饰、建筑空间、家具陈设、器物道具、人物设定包、士人/官员/民间人物服饰条目，都应按身份、场景和使用方式拆分，而不是只给笼统描述。

服饰类推荐结构：

- 少年/青年读书人：服饰、发式、鞋履、随身物、适用场景。
- 书院讲学者/文人：服饰、冠巾、讲学道具、语气边界、适用场景。
- 地方官员/正式身份：官服或常服、冠帽、腰带、印信、官署场景。
- 民间劳作者/旅人：劳动服饰、行旅道具、地域和阶层边界。
- 女性角色、儿童、僧道、军士等特殊身份：单独列明，不混用士人设定。

建筑与空间类推荐结构：

- 空间身份：书院、祠庙、官署、民居、洞穴、渡口、街巷等。
- 主时代：明确主时代；若为现存建筑，要区分初建、重修、现存展示。
- 使用场景：祭祀、讲学、夜读、审案、行旅、集市、传说相遇等。
- 固定陈设：门窗、案桌、灯盏、牌匾、屏风、石阶、岩壁等。
- 不可混用项：其他朝代构件、现代展示牌、影视化布景、旅游化新增设施。

器物与道具类推荐结构：

- 人物随身道具：随人物移动，如书卷、手稿、印章、书袋、伞、灯笼。
- 场景陈设：固定在空间中，如书桌、书架、香炉、砚台、灯盏、石阶。
- 身份边界：同一器物用于士人、官员、僧道、民间劳作者时应分别标注。
- 使用动作：读、写、携带、供奉、照明、通行、交易、礼仪等。
- 时代边界：注明可证时代、合理复原层和后世参照层。

每类都要写清：

- 服装：固定穿着，不归入随身道具。
- 发式/冠帽：角色识别元素，需符合时代。
- 人物随身道具：可移动物件，如书卷、手稿、印章、书袋。
- 不可混用项：其他朝代或现代元素，例如明清官帽、现代眼镜、近现代制服。

## 5. 朝代边界

涉及服饰、建筑、家具、文房、仪式、器物、称谓时，必须标注朝代边界。

写法建议：

- `era` 字段写主时代，如 `宋`、`唐`、`明清`。
- 如果引用后世材料补审美或陈设，必须在正文和可信度中写明“后世参照，不等同原貌”。
- 跨朝代复原不得直接写成某朝真实场景。
- 建筑和陈设条目要区分“现存展示状态”“历史原貌”“影视化复原”三层。
- 故事生成时，主角所处时代优先于素材来源时代。

## 6. 民间传说和志异边界

传说、志异、鬼怪、梦境、显灵、报恩、遇异等内容必须分层：

- 史料可证：人物、地名、著作、任职、文物遗存等可核资料。
- 地方传说：口述、景区说明、戏曲或民间文学版本。
- 文学志异：小说、笔记、传奇文本中的叙事结构。
- 创作可用：可用于故事气氛和结构，但不得冒充史实。

`### 待核实点` 中必须列出最少一个边界问题，例如最早出处、版本差异、实地位置、人物真实性、道具是否为后世附会。

## 7. 地方化人物故事关系

当甲方要求“某个人物在某地发生的事、产生的思想文化影响”时，条目不得只写人物传记，也不得把同省、同文化圈或后世影响误写成本人亲历。应新增或补充以下小节：

```md
### 地方化创作关系

- 直接事件｜地点：人物本人可核实的任职、居住、讲学、写作、会友、活动。
- 相关地点｜地点：祠庙、书院、纪念馆、碑刻、展陈、地方志记录等遗存或纪念空间。
- 思想文化影响｜地点：人物思想、著作、人格形象经由书院、学脉、教育、文旅阐释进入当地。
- 当代转化｜地点：当代研学、展陈、廉洁教育、城市文旅、非遗演出等使用方式。
- 不可写成｜地点：明确禁止写成的伪亲历、伪因果或待核实情节。
```

使用原则：

- 优先写“直接事件”；没有直接事件时，明确改写为“相关地点”“思想文化影响”或“当代转化”。
- `相关地点` 可以作为画面入口，但不能自动等同于人物到过该地。
- `思想文化影响` 可以作为地方化故事主轴，例如“某思想如何被当地书院、学脉、教育传统吸收”，但不能写成“人物本人在当地提出该思想”。
- `不可写成` 必须写清楚创作红线，尤其是名人、地方政府客户、研学和文旅项目。
- 生成故事时，标题、旁白、分镜和 GEARS 供稿均应保留关系类型，不得把“后世传播”改成“现场发生”。

例：周敦颐与长沙若缺少本人在长沙亲历的确证事件，应优先写“周敦颐思想经由岳麓书院和湖湘学脉在长沙被接受、阐释和传播”，而不是写“周敦颐在长沙讲学/悟道”。
