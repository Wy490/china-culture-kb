# 中国传统文化知识库 Agent 设计文档

日期：2026-05-27

## 概述

构建一个基于 MCP Server + Claude Code 的中国传统文化知识库 Agent 系统。MVP阶段实现核心流程（采集 → 核查 → 入库 → 匹配 → 补充），第二阶段扩展脚本生成功能。

## 决策记录

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 系统形态 | MVP优先，CLI形态 | 最快跑起来，后续再考虑Web |
| 存储方式 | Markdown文件 | 与现有项目一致，零基础设施依赖 |
| 来源类型 | B站视频、网络文章、书籍论文、口述传承 | 覆盖传统文化主要来源渠道 |
| 匹配方式 | LLM语义匹配 | Claude Code直接分析，灵活且无需额外索引 |
| 核实策略 | 混合：外部权威优先+内部互证补充 | 最全面的核实路径，记录核实方法 |

## 系统架构

### 组件关系

```
用户 → Claude Code（对话编排、语义分析） ↔ MCP Server（7个KB专用工具） ↔ 知识库（Markdown文件）
```

- **Claude Code**：交互入口和编排引擎，负责对话、LLM语义分析、决策调用工具
- **MCP Server**：提供知识库专用工具，处理文件读写、来源抓取、检索等操作
- **知识库**：现有Markdown文件结构（data/provinces/、sources/、indexes/）

### 数据目录结构

```
data/provinces/        ← 34个省级条目文件
data/regions/          ← 地区补充资料
sources/videos/        ← B站视频资料（BV号.md）
sources/articles/      ← 网络文章资料（URL标识.md）
sources/oral/          ← 口述传承资料（讲述人.md）
sources/books/         ← 书籍论文资料（书名.md）
sources/websites/      ← 网页来源记录
templates/             ← 条目模板
indexes/               ← 索引文件
notes/                 ← 临时笔记
scripts/               ← 生成的影视脚本（Phase 2）
```

## MVP Phase 1：MCP工具定义

### kb_fetch_video

从B站视频抓取内容。

**输入：** bv_id (BV号) 或 url (视频链接)

**输出：** 标题、UP主、发布时间、简介、字幕/弹幕摘要、视频中提及的故事名称列表

**实现：** 调用B站API获取视频信息；yt-dlp提取字幕；AI提取故事名称

### kb_fetch_article

从网页文章抓取内容。

**输入：** url (文章链接)

**输出：** 标题、作者、发布时间、正文(Markdown)、来源平台

**实现：** WebFetch抓取HTML → 转Markdown；记录来源元数据

### kb_verify_source

核查来源可信度。混合核实策略。

**输入：** source_type, source_url (可选), source_author (可选), claims (待核实内容)

**输出：** 可信度评级、核实方法标注、核查结果详情、待核实点清单

**核实流程：**

1. **外部权威检索**：Claude Code调用WebSearch/WebFetch搜索非遗名录、文化厅数据库、知网等权威来源 → 找到佐证标"外部核实"。kb_verify_source本身不做网络搜索，它接收Claude Code传入的外部核实结果作为输入的一部分。
2. **知识库内部互证**：用kb_match/kb_search找同故事版本、同类型条目 → 3条以上佐证标"内部互证"；1-2条标"内部部分佐证"
3. **无佐证**：标"存疑"，列出所有待核实点，建议用户后续寻找外部来源

**来源类型核查标准：**

| 来源类型 | 默认可信度 | 核查要点 | 必记录字段 |
|----------|-----------|----------|-----------|
| B站视频 | 待核实 | UP主资质、官方账号背书、评论区考证 | BV号、标题、UP主、发布时间、链接 |
| 网络文章 | 待核实 | 平台权威性、作者资质、引用来源 | URL、标题、作者、平台、发布时间 |
| 书籍论文 | 基本可靠 | 出版社/期刊权威性、ISBN/DOI | 书名/论文标题、作者、ISBN/DOI、出版社、出版年 |
| 口述传承 | 待核实 | 讲述人背景、地点吻合度、其他版本佐证 | 讲述人、讲述地点、时间、场景、记录人 |

### kb_add_entry

写入条目到省份文件。

**输入：** 完整条目字段（名称、省份、地区、类型、简介、故事梗概、文化意义、相关地点、关键词、来源、可信度、待核实点）

**输出：** 写入确认、文件路径

**行为：**
1. 验证字段完整性（必须包含所有CLAUDE.md要求的字段）
2. 确定对应省份文件
3. 按条目模板格式写入
4. 跨省份条目自动在多个文件创建链接条目
5. 更新索引

### kb_match

语义匹配知识库条目。

**输入：** story_text (用户上传的故事或观点文本), province_hints (可选，Claude Code分析后传入的地理线索), type_hint (可选，Claude Code分析后传入的类型判断)

**输出：** 匹配条目列表，每个含：条目名称、匹配原因、相似度评级、省份

**匹配流程：**
1. Claude Code先在对话中分析用户输入，提取4个维度：
   - 主题识别（孝道、忠义、爱情、因果等）
   - 地理线索（地名、省份、河流、山脉）
   - 人物识别（关键人物名）
   - 类型判断（神话传说/民间故事/节庆习俗等）
2. Claude Code调用kb_match，传入story_text及province_hints/type_hint
3. kb_match优先读取province_hints对应的省份文件，然后遍历其余省份文件，返回全部条目内容
4. Claude Code在对话中对返回的条目内容做语义对比，输出匹配结果

**相似度评级：**
- 高度匹配：同一故事/人物/地点，核心叙事基本一致
- 中度匹配：主题相同、有共同元素，叙事差异较大
- 低度匹配：仅类型或主题相关，内容差异显著

### kb_search

按条件检索知识库。

**输入：** keywords, type (可选), province (可选), region (可选)

**输出：** 匹配条目列表，含完整字段摘要

**实现：** 遍历指定省份文件 → 关键词文本搜索 → 返回结果

### kb_supplement

补充相关知识库内容。三维度补充。

**输入：** entry_name 或 story_text

**输出：** 补充内容，按三个维度组织

**三维度补充：**

1. **版本差异**：同一故事在不同省份的版本对比。用kb_search同关键词跨省检索。
2. **同地同类**：同一省份同一类型的其他条目。用kb_search按省份+类型检索。
3. **关联网络**：关联的人物、地点、事件。用kb_search关键词交叉检索。

**补充结果格式：**
```
### 补充内容
**版本差异**：[[浙江]]版本中白蛇修炼千年而非五百年
**同地同类**：江苏另有"柳毅传书"民间故事，同属爱情传说类
**关联网络**：相关人物—许仙；相关地点—西湖断桥、金山寺
```

## 核实结果记录格式

每个条目的可信度评级附带核实方法标注：

```
### 可信度
基本可靠

### 核实方法
外部核实：非遗名录"白蛇传"条目印证
内部互证：江苏、浙江、安徽3省份均有类似版本

### 待核实点
- 该版本与主流版本的差异原因待查
- 讲述人提到的具体地名需核实
```

## Phase 2：脚本生成（架构预留）

### kb_generate_script

从知识库条目生成AI影视脚本。

**输入：** entry_names (1个或多个知识库条目), script_type, target_duration

**输出：** Markdown格式的影视脚本文件，写入scripts/目录

**脚本类型：**

| 类型 | 适用条目类型 | 结构特点 |
|------|-------------|---------|
| 纪录片脚本 | 非遗、传统工艺 | 旁白+画面+采访 |
| 短剧脚本 | 民间故事、地方掌故 | 场景+对话+动作 |
| 动画脚本 | 神话传说 | 画面描述+旁白 |
| 文化解说脚本 | 节庆习俗、饮食文化 | 主题讲解+案例 |

**脚本模板结构：**

```
# {{条目名称}} — {{脚本类型}}脚本

## 元信息
- 来源条目：[[条目1]]、[[条目2]]
- 脚本类型：短剧/纪录片/动画/文化解说
- 目标时长：{{分钟数}}
- 文化背景说明：{{从知识库补充的文化背景}}

## 角色表
- {{角色1}}：{{描述}}，来源：[[条目]]

## 场景列表
### 场景1：{{场景名}}
- 地点：{{地点描述}}
- 时间：{{时间设定}}
- 画面：{{画面描述}}
- 旁白/对话：{{具体内容}}
- 文化注释：{{需要注明的文化细节}}

## 文化注释总汇
- {{需要特别注意的文化细节和准确性要点}}

## 来源溯源
- 本脚本基于知识库条目：[[条目名]]
- 版本差异说明：{{如有多版本，说明选择依据}}
```

关键设计原则：脚本每个元素都标注知识库来源条目，确保文化准确性可追溯。

Phase 2仅做架构预留，MVP阶段不实现。脚本模板和类型选项待Phase 2启动时细化。

## 技术实现要点

### MCP Server实现

- 使用TypeScript + MCP SDK实现MCP Server
- Server进程独立运行，Claude Code通过stdio连接
- 工具参数验证：kb_add_entry必须验证所有必填字段
- 文件操作：读写Markdown文件，使用Node.js fs模块
- B站API：调用bilibili API获取视频信息，yt-dlp提取字幕
- 网页抓取：使用内置WebFetch或cheerio解析HTML

### Claude Code集成

- MCP Server配置在.claude/settings.json中
- Claude Code负责对话编排和LLM语义分析
- kb_match的语义比较由Claude Code在对话中完成（读取文件内容后直接分析）
- 核查逻辑：kb_verify_source返回结构化核查结果，Claude Code负责判断和标注

### 知识库文件约定

- 省份文件格式：按CLAUDE.md定义的条目字段
- 跨省份条目：在多个省份文件创建简要条目，互相链接
- 索引文件：indexes/目录维护类型索引、关键词索引、地区索引
- 来源文件：sources/目录按类型分目录，每个来源独立.md文件
